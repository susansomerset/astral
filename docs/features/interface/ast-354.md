# AST-354 — Update UI Job Tables

## Plan

### Problem

The In Review, Skipped, and Recommended job tables showed full-label column headers (or raw
vector codes without labels), letter-text grades, no `latest_score` column, and no sort
controls — causing job titles to wrap to 6+ lines and grade values to be harder to scan at a
glance.

### Changes

**Column headers** — derived from `candidate_data.artifacts[<rubricKey>]` (`code` for the th
text, `label` for the `title` tooltip). Falls back to raw vector value if the rubric artifact is
absent or empty. Mapping: `joblist_grades` → `joblist_rubric`, `jd_grades` → `jobdesc_rubric`,
`do_grades` → `do_rubric`, `get_grades` → `get_rubric`, `like_grades` → `like_rubric`.

**Grade dots** — replaced letter-text cells with `.grade-dot .dot-{a|b|c|d|f|x}` spans (existing
CSS). Missing grade falls back to `—`.

**`latest_score` column** — uses the direct DB column (`SELECT *` already includes it; no API
change needed). Shown only when at least one job in the section has a non-null score. Formatted
as `.toFixed(2)` (0.00 – 10.00 scale). Column sits between grade columns and the date column.

**Click-sortable columns** — all data columns sortable client-side. Sort state is per-section
(`Record<string, SortState>`) for InReview/Skipped; single `SortState` for Recommended.
Ascending/descending toggle on re-click. Active column shows ▲/▼ indicator (10px, inline).
Nulls sort last for `latest_score`. Grade columns sort A→F ascending, unknown/missing last.

### Files Changed

| File | Change |
|---|---|
| `src/ui/frontend/src/pages/JobsInReview.tsx` | Rubric headers, grade dots, `latest_score`, per-section sort state |
| `src/ui/frontend/src/pages/JobsSkipped.tsx` | Same, plus preserves bulk-retry checkbox logic |
| `src/ui/frontend/src/pages/JobsRecommended.tsx` | Same, single-table sort state, `like_rubric` lookup |
| Backend / API | No changes — `latest_score` is a direct `job` table column in `SELECT *` |
| CSS | No changes — `.grade-dot`, `.dot-*`, `.sortable` all already in `App.css` |
